## 6) Senior C review checklist (what to look for)

- Use of uninitialized variables
- Out-of-bounds reads/writes

- Misuse of `sizeof` (pointer vs array)
- Violations around strict aliasing
- Lifetime issues (returning pointers to stack memory)
- `memcpy/memmove/strncpy` misuse

- Every allocation has one clear owner
- All early returns free resources
- No double-free, no use-after-free
- Clear “init”/“deinit” symmetry per module

### API & Design
- Headers expose minimal surface area
- Opaque structs where appropriate

- Explicit error returns (`bool`, error codes) and consistent conventions

### Build & Portability
- C standard targeted (C11/C17)

### Performance
- Avoid per-frame allocations
- Avoid loading textures/fonts inside the main loop
- Batch draws; limit state changes
- Profile before optimizing
